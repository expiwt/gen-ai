## 2. Где модель ошибалась

### Пример 1: Потеря контекста при фрагментации
Изначально интервью было разрезано на 5 частей. LLM в каждом фрагменте определяла эксперта по-разному:
- Фрагмент 1 → «Михаил Хазин»
- Фрагменты 2-3 → «неизвестно» (потому что в начале этих частей отвечал ведущий)
- Фрагменты 4-5 → «Михаил»

**Проблема:** LLM не видит контекст предыдущих фрагментов и воспринимает ведущего как отдельного "эксперта".

**Решение:** склеить фрагменты в один файл — после этого эксперт стабильно определён как «Михаил Хазин», извлечено 24 тезиса вместо 7-12 на фрагмент.

### Пример 2: Заниженные оценки обоснованности
Средние оценки по аспектам:
| Аспект | Средняя оценка |
|--------|---------------|
| Новизна | 3.0 / 5 |
| Обоснованность | 1.3 / 5 |
| Практичность | 1.5 / 5 |
| Реалистичность | 1.9 / 5 |

**Причина:** DeepSeek оценивает тезисы Хазина (субъективные прогнозы о крахе банков и геополитике) как слабо обоснованные. Это справедливо для экономического форкаста, где сложно привести объективные доказательства.

**Искажение:** Модель путает «отсутствие доказательств в формате академической статьи» с «отсутствием обоснования» — в интервью Хазин приводит аргументы, но в разговорном, а не научном стиле.

### Пример 3: Работа LLM-as-Judge
При `overall_score = 0.65 < 0.7` был автоматически перезапущен Reduce с улучшенным промптом. После повторного Reduce Judge был также перезапущен, но оценка осталась 0.65.

**Анализ:** DeepSeek последовательно консервативен в роли судьи. Основные замечания:
- «Недостаточная полнота охвата»
- «Оценки обоснованности занижены»

Это не ошибка пайплайна, а особенность используемой модели-судьи.

### Autodiscovery (раунд 2.5)

После аспектного анализа по фиксированному списку был запущен Autodiscovery — модель сама предложила дополнительные аспекты:

| Доп. аспект | Зачем |
|------------|-------|
| Геополитическая обоснованность | Оценка геополитических утверждений, которые составляют основу интервью |
| Экономическая логика | Проверка внутренней связности экономических рассуждений |
| Историческая аналогия | Оценка корректности исторических параллелей (Великая депрессия и др.) |

**Вывод:** Autodiscovery полезен — он выявил аспекты, релевантные для данного типа интервью, которые не входили в универсальный фиксированный список. В production стоило бы использовать гибридный подход: фиксированный список как база + autodiscovery для специфики предметной области.

## 3. Что бы изменили в production

### Оставить как есть
- **IE с `check_quotes`** — полезная защита от галлюцинаций (0% ghost-цитат)
- **Аспектный анализ** — хорошо структурирует оценки
- **Map-Reduce** — даёт компактную сводку из 5 тезисов

### Переделать
1. **REDUCE промпт** — сделать более строгим (с具体要求ми к подсчёту средних и согласованности). Текущая версия позволяет LLM самой интерпретировать цифры, что ведёт к расхождению.

2. **Обработка длинных текстов** — при `text[:5000]` терялось 95% интервью. Нужно либо увеличить контекст, либо добавить Map-Reduce на этапе IE для длинных текстов.

3. **LLM-as-Judge** — использовать несколько судей (ансамбль моделей) или human-in-the-loop для валидации.

### Добавить
- **Кэширование (раунд 6)** — повторные запросы с теми же данными могут кэшироваться, экономя ~90% стоимости. DeepSeek поддерживает prompt caching.
- **Multi-doc (раунд 7)** — если интервью несколько, добавить cross-source consolidation.
- **A/B тестирование** — сравнить DeepSeek vs GPT-4o-mini на одном датасете для оценки bias судьи.
